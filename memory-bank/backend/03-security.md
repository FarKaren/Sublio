# 04 — Security

## Threat model

```
Threats                            Where it occurs        How we protect
─────────────────────────────────────────────────────────────────────────
Unauthorized access                All endpoints          JWT RS256 in Gateway
DDoS / abuse                       Gateway                Rate limiting per IP
Malicious file upload              media-service          Magic bytes validation
Path traversal                     media-service          UUID names, path check
SQL injection                      subtitle-service       Parameterized queries (JPA)
Secret leakage via env             All services           Docker secrets / .env not in code
Compromise of one service          Internal network       Isolation via Docker network
Password brute force               auth-service           bcrypt + rate limit on /login
MITM                               External traffic       TLS everywhere
Hung Python process                transcription-worker   Task timeout + Docker limits
XSS                                frontend               CSP headers in gateway
CSRF                               frontend               JWT (not cookie) → no CSRF
```

## Layer 2: Auth Service (Go)

```
Passwords:
  └── bcrypt(password, cost=12)
      The password itself is never stored

JWT key pair:
  ├── private.pem  → stored in auth-service (signs tokens)
  └── public.pem   → mounted in api-gateway (verification only)
  Private key NEVER leaves auth-service

RefreshToken:
  ├── generated as crypto/rand (32 bytes)
  ├── stored as SHA-256 hash in PostgreSQL
  └── on use → old one is invalidated, new one is issued

Brute force protection on /login:
  └── rate limit: 10 req/min per IP at gateway level
```

## Layer 3: Media Service (Go)

```
Uploaded file validation:

  1. Size:
     Content-Length checked BEFORE reading the body
     Maximum: 2GB (configurable)

  2. Magic bytes (first 12 bytes):
     MP4:  bytes [4:8] == "ftyp"
     MKV:  [0:4] == {0x1A, 0x45, 0xDF, 0xA3}
     AVI:  [0:4] == "RIFF" AND [8:12] == "AVI "
     If no match → 415 Unsupported Media Type

  3. Filename:
     Original name → IGNORED
     On disk: /data/processed/{uuid}/video.mp4
     UUID generated by crypto/rand (not predictable)

  4. Path traversal:
     folderId from URL validated with regexp: ^[0-9a-f-]{36}$
     If not UUID format → 400

  5. Other users' files:
     On GET /video/:folderId → job-service checks that
     folderId belongs to userId from JWT
```

---

## Layer 4: Job Service (Go)

```
Ownership check:
  └── each job stores user_id
      On GET/DELETE /jobs/:id → compared with userId from JWT
      Foreign jobId → 403 Forbidden (not 404, to avoid revealing existence)

SSE protection:
  └── SSE connection is bound to a specific jobId + userId
      Cannot subscribe to someone else's progress

Job timeout:
  └── if job is in TRANSCRIBING status for > 30 min → automatically ERROR
      Protection against hung workers
```

## Layer 5: Subtitle Service (Kotlin)

```
SQL injection:
  └── Spring Data JPA + Parameterized queries
      No String concatenation in SQL

Input validation:
  └── srtPath is checked to start with /data/processed/
      Cannot escape the shared volume boundaries

Availability:
  └── port 8084 is open ONLY inside Docker network sublio_net
      Unreachable from outside even if port is known
```

---

## Layer 6: Transcription Worker (Python)

```
Isolation:
  ├── no open HTTP ports (only outbound Redis)
  ├── runs as non-root user in Docker
  ├── read-only filesystem except /data/processed/
  └── Docker resource limits:
        --cpus=2
        --memory=4g

Input data:
  └── filePath from Redis message is validated:
      ├── path starts with /data/processed/
      └── file exists before launching Whisper
```

## Infrastructure

```
PostgreSQL:
  ├── password generated on first run (docker secret)
  ├── each service has its OWN user with minimum privileges:
  │     auth_user    → SELECT/INSERT/UPDATE on users, refresh_tokens
  │     job_user     → SELECT/INSERT/UPDATE on jobs
  │     subtitle_user → SELECT/INSERT on subtitles, subtitle_entries
  └── data encrypted at-rest (PostgreSQL pgcrypto or disk encryption)

Redis:
  ├── requirepass (AUTH password)
  ├── bind 127.0.0.1 (only inside Docker network)
  └── TLS for connections from services

Secrets (not stored in code or .env in repo):
  ├── DB_PASSWORD
  ├── REDIS_PASSWORD
  ├── JWT_PRIVATE_KEY
  └── JWT_PUBLIC_KEY
  Stored in: docker-compose secrets or external secret manager

.gitignore must include:
  ├── *.pem
  ├── .env
  └── /infra/certs/
```
